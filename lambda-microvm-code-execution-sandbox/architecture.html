Lambda MicroVM Code Execution Sandbox

Isolated Firecracker MicroVM that runs untrusted Python over HTTP — snapshot-resumed, auth-gated, auto-suspending

Isolation & Security

  • • Firecracker hardware-level VM isolation per MicroVM
  • • Untrusted code runs in a python3 subprocess
  • • 30-second execution timeout enforced
  • • HTTP API gated by an auth token (X-aws-proxy-auth)
  • • Each request writes to an ephemeral temp file, then unlinked

Runtime & Lifecycle

  • • Image built server-side from a Dockerfile, then snapshotted
  • • Resumes from snapshot in under a second
  • • Auto-suspends after 15 min idle; auto-resumes on next request
  • • Terminates after 5 min in suspended state
  • • Lifecycle hooks on :9000 · structured logs to CloudWatch

Execution API

  • • POST /execute with a JSON { "code": "..." } body
  • • Returns stdout, stderr, exit_code, success
  • • GET / health check reports service + microvm_id
  • • Single MicroVMBuildRole (build + execution)
  • • Python 3.14-slim base image · us-east-2
