Lambda MicroVM from ECR

Build locally, push to Amazon ECR, and have Lambda pull your image during the server-side MicroVM build

Build & Deploy

  • • Build locally/CI: docker buildx --platform linux/arm64
  • • Push the image to Amazon ECR (microvm-from-ecr:latest)
  • • Upload a thin zip — just FROM <ecr-uri> — to S3
  • • create-microvm-image: Lambda pulls the ECR image during build
  • • Built on the al2023-0 base, then Firecracker-snapshotted

IAM & Access

  • • One MicroVMRole assumed at build and at run
  • • AmazonEC2ContainerRegistryReadOnly — ECR pull at build
  • • AmazonS3ReadOnlyAccess — reads the thin zip
  • • CloudWatchLogsFullAccess — execution-time logging
  • • HTTP endpoint gated by an auth token (:8080)

Runtime

  • • Minimal app.py — Python stdlib only, no dependencies
  • • GET / health check on port 8080
  • • Lifecycle hooks on port 9000 (/run delivers microVmId)
  • • Auto-suspends after 15 min idle; resumes in under a second
  • • Best for complex builds, private base images, CI/CD · us-east-2
