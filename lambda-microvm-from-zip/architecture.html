Lambda MicroVM from Zip

Zip a Dockerfile and app code to S3, and let Lambda build the image server-side, snapshot it, and run it

Build & Deploy

  • • Zip src/ (Dockerfile + app.py) and upload to S3
  • • create-microvm-image builds the Dockerfile server-side
  • • Built on the al2023-0 base image — no local Docker needed
  • • App signals readiness via the /ready hook, then snapshot
  • • run-microvm resumes from snapshot in under a second

IAM & Access

  • • One MicroVMRole assumed at build and at run
  • • AmazonS3ReadOnlyAccess — reads the zip during build
  • • CloudWatchLogsFullAccess — execution-time logging
  • • No ECR or container registry required
  • • HTTP endpoint gated by an auth token (:8080)

Runtime & When to use

  • • Minimal app.py — Python stdlib only, no dependencies
  • • GET / health check on port 8080
  • • Lifecycle hooks on port 9000 (/run delivers microVmId)
  • • Auto-suspends after 15 min idle; resumes on demand
  • • Best for simple apps and quick iteration · us-east-2
