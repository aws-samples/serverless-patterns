# Handover: strands-agentcore-smithy

Build a serverless AI agent using AWS Bedrock AgentCore Gateway with a Smithy model target type. The agent (Claude 3 Sonnet) calls another model (Claude 3 Haiku) as a tool via the Bedrock Runtime Converse API through the gateway.

## Architecture

```
User → Agent Lambda (Strands SDK + Bedrock Claude 3 Sonnet)
     → Cognito JWT authentication (ID tokens)
     → AgentCore Gateway (MCP protocol, CUSTOM_JWT authorizer)
     → Smithy Model Target (official Bedrock Runtime model from S3)
     → Bedrock Runtime Converse API → Claude 3 Haiku
```

## What is a Smithy Target?

Smithy targets connect AgentCore Gateway to services defined using Smithy API models. AgentCore translates incoming MCP requests into API calls to those services and handles response formatting.

Key constraints:
- For AWS services, you MUST use the official Smithy model from the AWS API Models repo
- Custom Smithy models cannot resolve AWS service endpoints (missing `aws.api#service` trait and full endpoint rules)
- Only RestJson protocol is supported (services using `awsJson1_0` like DynamoDB won't work)
- Max model size: 10MB
- No streaming operations

## Deployment

One command:

```bash
bash scripts/deploy.sh
```

The script:
1. Validates the CloudFormation template
2. Downloads the official Bedrock Runtime Smithy model from GitHub and uploads to S3
3. Creates/updates the CloudFormation stack
4. Packages Lambda code (two-step pip3 install for binary + pure Python packages)
5. Deploys Lambda code (S3 fallback if >50MB)
6. Creates a Cognito test user
7. Generates `scripts/test.sh` with baked-in values

## Testing

```bash
./scripts/test.sh "Ask Haiku to write a haiku about clouds"
./scripts/test.sh  # uses default prompt
```

## CloudFormation Resources

| Resource | Type | Purpose |
|----------|------|---------|
| `AgentCoreGateway` | `AWS::BedrockAgentCore::Gateway` | MCP gateway with CUSTOM_JWT auth |
| `SmithyTarget` | `AWS::BedrockAgentCore::GatewayTarget` | Smithy model target pointing to Bedrock Runtime via S3 |
| `CognitoUserPool` | `AWS::Cognito::UserPool` | User authentication |
| `CognitoUserPoolClient` | `AWS::Cognito::UserPoolClient` | Client for USER_PASSWORD_AUTH flow |
| `AgentLambdaFunction` | `AWS::Lambda::Function` | Strands SDK agent (Python 3.12, x86_64) |
| `AgentLambdaRole` | `AWS::IAM::Role` | Lambda execution role (Bedrock + AgentCore + Logs) |
| `GatewayExecutionRole` | `AWS::IAM::Role` | Gateway role (4 ARN patterns + Bedrock Runtime + S3) |

## Key Lessons Learned

### 1. Must Use Official AWS Smithy Models for AWS Services
A custom Smithy model with just `aws.auth#sigv4` and `aws.protocols#restJson1` fails silently — the gateway needs the full `aws.api#service` trait, endpoint rule sets with partition data, and FIPS/DualStack logic. Only the official models from `github.com/aws/api-models-aws` have this.

### 2. Use Converse, Not InvokeModel
`InvokeModel` uses a `Blob` with `httpPayload` trait for the request body. When the gateway converts this to an MCP tool, the blob field disappears — there's no input parameter for the body. `Converse` has structured `messages` and `modelId` fields that map cleanly to MCP tool inputs.

### 3. awsJson1_0 Services Don't Work
DynamoDB, SQS, SNS, and other services using `awsJson1_0` protocol are incompatible with Smithy model targets (which only support `restJson1`). Bedrock Runtime uses `restJson1` so it works.

### 4. System Prompt Guides Tool Usage
The official Smithy model exposes ~10 tools with complex schemas. Without a system prompt, the LLM guesses wrong model IDs and message formats. The system prompt in `agent_processor.py` tells the agent to use Converse with the correct model ID and message structure.

### 5. Broad Bedrock Permissions Required
`bedrock:InvokeModel` on `foundation-model/*` isn't enough — cross-region inference profiles use different ARN patterns. Use `Resource: '*'` with all four Bedrock actions.

### 6. CloudFormation Property Casing Matters
- `CustomJWTAuthorizer` (all caps JWT) inside `AuthorizerConfiguration`
- `RoleArn` not `ExecutionRoleArn`
- `AllowedAudience` not `Audience`
- `CredentialProviderConfigurations` (array) not `CredentialProviderType` (top-level)
- `SmithyModel.S3.Uri` not `SmithyModel.S3Configuration.Uri`

### 7. Lambda Packaging Two-Step pip Install
`--only-binary=:all:` with `--platform` silently skips pure Python packages. Must run a second `pip3 install --no-deps` for requests, urllib3, PyJWT, etc.

### 8. Agent Code is Target-Agnostic
All code in `src/agent/` and `src/shared/` connects via MCP and doesn't reference any specific target. Same code works with API Gateway, Smithy, or Lambda targets.

## Prerequisites

- AWS CLI v2
- Python 3.12+
- pip3
- AWS account with Bedrock and AgentCore enabled in `us-east-1`
- Bedrock model access for Claude 3 Sonnet and Claude 3 Haiku
